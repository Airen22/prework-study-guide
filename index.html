HTML

  • The head element contains informaiton about the wepage such as metadata for the browser to use
  • The body element represents the visible content shown to the user.
  • [h1]-[h6] are heading levels
  • [p] indicates paragraph or block of text to wrap text
  • [ul], [ol], [li] are lists; unordered, ordered, list items respectively
  • [div] and [span] create division in the page
  • [br] creates a line break
  • [a] "anchor"; creates lins to the same webpage or other webpages

CSS

  • margin: space around the outside of an element
  • padding: space around the content inside an element
  • inline CSS: used to style elements directly in HTML (e.g. "style")
  • internal style sheet: contains CSS rules for webpage in head section of HTML file by embedding entire style sheet directly in HTML file
  • external CSS style sheet: preferred; allows developers to keep all CSS rules in seperate file for easier design changes and then link it into the head data
  • CSS rules have a selector (whate the rule applies to) and declaration (rules)
  • Declarations contain the CSS property and a value (e.g. color: blue)

Git

  • [pwd]"" displays currenty working directory ("print working directory")
  • [ls]: displays list of files and folders in the current directory
  • [cd]: "change directory"; navigates b/w directories in the path
  • [cd..]: navigates backwards, one level at a time through the path of your directories
  • [mkdir]: creates a new directory (folder)
  • [touch]: creates a file inside a directory
  • [git status]: checks what branch we are currently on
  • [git checkout -b branch-name]: creates a new branch and switches to it
  • [git add -A]: stages all changes to commit and push
  • [git commit -m "comment"]: associates a message with commit

JavaScript

  • A variable is a named container that allows us to store data in our code; "var"
  • Control flow is the order in which a compter executes code in a script
  • semicolon (;): used to end statement. generally used to seperate statements on the same line
  • naming variables: case sensitive; usually start with letter, "_", or "$"
  • String variable: text enclided in single quote marks (let myVaraible = 'x';)
  • Number: number; no quote marks (let myVariable = 10;)
  • Boolean: true/false value where true and fulse do not need quote marks (let myVariable = true;)
  • Array: allows storaage of multiple values in a single reference (let myVariable = [1, 'Bob', 'Steve', 10];
  • Object: Everything/Anything in JavaScript can be stored in a variable (let myVariable = document.querySelector('h1');)
  • Comments: snipets of text ignored by the browser (/* comment */) OR (// comment; if there are no linebreaks in code)
  • Operator: mathematical sybmbol that produce a result based on two values (or variables
  • +, -, *, /: addition, subtraction, multiplication, divisiton of two numbers togenter or two strings
  • =: assignment; assigns a value to a variable
  • ===: Strict equalty; perfomrs a teset to see if two values are equal and returns true or false
  • !, !==: Not, Does-not-equal; returns logically opposite value of what it precedes (true --> false)
  • if...else: if () is the test; it true first block of code runs; else code runs if not true
  • Functions: typically a variable(); defines a body of code that executes when function name is called
  • arguments: bits of data needed for a function to run; go inside (), separated by commas if there is more than one argument.
  • return: tells browser to return the result variable out of the funciton so it is available to use
  • event: code structures that listen for activity in the browser and run code in response. (addEventListener"click")
  • JavaScript elements are added UNDER the footer of the .html
  • falsy: value that is one of the following values 0, -0, On, "", null, undefined, NaN, False(boolean)
  • truthy: all other values that are not falsy
